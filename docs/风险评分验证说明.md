# 环境风险评分的单调性与敏感性验证

## 1. 验证目的

当前 `risk.score` 是规则生成的相对风险指数，不是根据真实跌倒结局训练得到的跌倒概率。本验证用于回答两个工程问题：

1. 当单一危险证据增强时，风险分是否保持不下降；
2. 当顶层分量权重在小范围内变化时，场景得分、等级和危险排序是否稳定。

验证直接向 `analyze_bisenet` 输入受控的分割 mask 和 YOLO 框，因此只评价评分公式，不评价 BiSeNet、YOLO 的识别准确率。

## 2. 运行方式

在项目根目录执行：

```bash
python tools/validate_risk_scoring.py
```

默认输出到：

```text
validation/scoring_validation/
```

可选参数：

```bash
python tools/validate_risk_scoring.py \
  --output-dir validation/scoring_validation \
  --weight-deltas -0.2 -0.1 0.1 0.2 \
  --boundary-margin 3
```

如果运行环境没有 Matplotlib，可以添加 `--no-plots`，只生成 JSON 和 CSV。

## 3. 单调性验证场景

工具按“由安全到危险”的顺序生成以下八组受控场景：

| 序列 | 唯一变化的变量 | 预期结果 |
| --- | --- | --- |
| 障碍向通道中心移动 | 同一检测框的横向位置 | 分数不下降 |
| 障碍面积 | 同一障碍的框面积 | 分数不下降 |
| 障碍数量 | 通道内非重叠检测框数量 | 分数不下降 |
| 通道变窄 | 可通行区域相对宽度 | 分数不下降 |
| 湿滑面积 | 通道内湿滑 mask 面积 | 分数不下降 |
| 台阶面积 | 台阶/门槛 mask 面积 | 分数不下降 |
| 光照变暗 | 均匀图像亮度 | 分数不下降 |
| 水渍置信度 | 固定 `liquid_spot` 框的置信度 | 分数不下降 |

相邻点只要出现后一个场景更危险、风险分却更低，就记录为一次单调性违反。分数相同允许存在，但会记录为平台区间。

## 4. 权重敏感性分析

当前顶层分量为：

| 分量 | 当前最高分 |
| --- | ---: |
| 通道阻塞 | 45 |
| 台阶/门槛 | 25 |
| 湿滑表面 | 20 |
| 低照度 | 10 |

敏感性分析采用单因素扰动：每次只将一个分量调整 `-20%`、`-10%`、`+10%` 或 `+20%`，其他分量按比例缩放，使总分仍为 100。每组权重均在九个固定场景上重新评分。

报告统计：

- 平均和最大分数变化；
- 低、中、高、紧急等级变化次数；
- 距离 25、50、75 分界线超过 3 分的非边界场景跨级次数；
- 是否发生跨越两个等级；
- 新旧场景排序的 Spearman 相关系数。

当前内部工程验收条件为：最大绝对分数变化不超过 5 分、最低排序相关系数不低于 0.95、非边界场景不跨级、任何场景不跨越两个等级。这些条件只用于发现评分公式回归，不属于临床有效性标准。

## 5. 2026-07-23 首次结果

首次运行结果如下：

- 8 组单调性序列全部通过，单调性违反数为 0；
- 14 个场景、16 组权重扰动中，最大绝对分数变化为 3.55 分；
- 共发生 13 次风险等级变化，但全部来自距离 25、50、75 分界线不超过 3 分的边界场景；
- 非边界场景跨级数为 0，跨越两个等级的次数为 0；
- 最低 Spearman 排序相关系数为 0.965。

这些结果说明当前公式在本组受控变量下具备基本单调性，顶层权重发生不超过 20% 的变化时，风险等级相对稳定。但九个合成场景规模较小，结果不能替代真实家庭场景或专家标注校准。

## 6. 验证发现的三个边界

### 6.1 通道中心位置存在平台

同一障碍逐步向通道中心移动时，风险分为：

```text
6.80 → 8.82 → 13.37 → 15.85 → 15.85
```

最后两个位置得分相同。当前公式能感知障碍是否进入通道以及通道占用面积，但没有连续的“距离通道中心线”权重。该结果属于能力边界，不属于单调性错误。

### 6.2 空通道仍有 6.80 分基础风险

完全可通行、无障碍、正常光照的受控图像仍得到 6.80 分。原因是通道代理区域自身具有远窄近宽的透视形状，而最窄宽度目前直接除以整幅图像宽度，产生了非零的“通道较窄”证据。

后续应考虑将实际可通行宽度与同一行的透视通道期望宽度比较，而不是直接与整幅图像宽度比较。本次验证暂不修改正式公式，以保留问题证据。

### 6.3 台阶分量存在阶跃和平台

只要出现一个有效台阶连通区域，`step_count / 3` 就会立即产生约三分之一的台阶证据；较小面积范围内随后出现平台。这符合当前离散计数公式，但可能导致小面积误检被过度放大，需要结合台阶类别较低的验证集准确率进一步处理。
